## Typical SDLC Phases

### 1. Discovery & Research
**Who:** Product Manager
**Duration:** Variable (1-5 days typical)
- Customer feedback analysis
- Competitive research
- Problem validation
- Initial scoping

### 2. Requirements & PRD
**Who:** Product Manager
**Duration:** 1-3 days
- Write PRD using `/prd` skill
- Define user stories
- Specify acceptance criteria
- Run QA checks

### 3. Design
**Who:** Designer
**Duration:** 3-10 days depending on complexity
- Review PRD
- Create mockups/prototypes
- Design review with PM
- Iterate based on feedback

### 4. Engineering Planning
**Who:** Engineering Lead + Team
**Duration:** 1-2 days
- Technical design
- Break into tasks
- Estimate effort
- Identify dependencies

### 5. Development
**Who:** Engineers
**Duration:** Variable (1-4 weeks typical)
- Implementation
- Code review
- Unit tests
- Integration

### 6. QA Testing
**Who:** QA Engineer (or Engineers)
**Duration:** 2-5 days
- Test against acceptance criteria
- Regression testing
- Bug fixes

### 7. Deployment
**Who:** Engineering
**Duration:** 1 day
- Staging deployment
- Production deployment
- Monitoring

### 8. Post-Launch
**Who:** Product Manager, Support
**Duration:** Ongoing
- Monitor metrics
- Collect feedback
- Iterate

## Key Milestones

| Milestone | Description | Required Artifacts |
|-----------|-------------|-------------------|
| **PRD Complete** | Requirements finalized | PRD with QA score ≥ 7.0 |
| **Design Complete** | Mockups approved | Figma files, design review sign-off |
| **Dev Complete** | Code merged to main | All tests passing, code reviewed |
| **QA Complete** | Testing finished | QA test report, all blockers resolved |
| **Shipped** | In production | Deployed, metrics tracking |
