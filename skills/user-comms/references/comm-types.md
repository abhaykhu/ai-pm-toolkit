# Communication Types Reference

## Changelog Entry

**Tone**: Informative, concise
**Length**: 50-200 words per entry

### Template
```markdown
## [Feature/Fix Name]
[One-line summary of what changed and why it matters to the user]

[Optional: 2-3 bullet points with details]

[Optional: link to docs or help article]
```

## Email Announcement

**Tone**: Engaging, value-focused
**Length**: 150-300 words

### Template
```markdown
**Subject:** [Benefit-driven subject line]
**Preview text:** [First line that shows in inbox preview]

Hi [name/there],

[Opening hook — what's new and why they should care]

[Key details — what it does, how it helps them]

[How to get started / CTA]

[Optional: link to blog post or docs]

[Sign-off]
```

## In-App Notification

**Tone**: Brief, actionable
**Length**: 20-50 words

### Template
```markdown
**Title:** [Short, clear headline — max 60 chars]
**Body:** [What's new + what they can do — max 120 chars]
**CTA Button:** [Action text — max 20 chars]
**Link:** [Where the CTA goes]
```

## Blog Post

**Tone**: Engaging, storytelling
**Length**: 500-1500 words

### Template
```markdown
# [Title — benefit or outcome driven]

[Hook paragraph — why should the reader care?]

## The Problem / Context
[Set up the situation]

## What's New
[Describe the solution/feature/update]

## How It Works
[Walkthrough with examples or screenshots]

## What This Means for You
[Concrete benefits by use case]

## Getting Started
[How to try it, links to docs]

## What's Next
[Tease future plans, invite feedback]
```

## Social Media Posts

### Twitter/X
**Length**: Max 280 characters
**Tone**: Punchy, conversational

```markdown
**Tweet:** [Announcement in ≤280 chars with key value prop]
**Thread (optional):**
1/ [Hook]
2/ [Detail]
3/ [CTA with link]
```

### LinkedIn
**Length**: 150-300 words
**Tone**: Professional, thought-leadership angle

```markdown
[Hook line that stops the scroll]

[Context — why this matters in the industry]

[What we did/launched]

[Impact or early results]

[CTA or question to drive engagement]

[Relevant hashtags]
```

## Support Response Template

**Tone**: Empathetic, helpful, solution-oriented
**Length**: 50-150 words per template

### Template
```markdown
# [Scenario Name]

**When to use:** [trigger condition]

---

Hi [name],

[Acknowledge their issue/question]

[Solution or explanation]

[Next steps or what to expect]

[Offer further help]

Best,
[Team/Name]
```

## Migration / Breaking Change Notice

**Tone**: Clear, urgent but not alarming, action-oriented
**Length**: 200-500 words

### Template
```markdown
# [Action Required] [What's Changing]

**Effective date:** [date]
**Who's affected:** [which users/plans/integrations]

## What's Changing
[Clear explanation of the change]

## Why
[Brief rationale]

## What You Need to Do
1. [Step-by-step migration instructions]
2. [Step]

## Timeline
| Date | What Happens |
|------|-------------|

## What If I Don't Act?
[What happens if they don't migrate]

## Need Help?
[Support channels, office hours, migration assistance]
```
